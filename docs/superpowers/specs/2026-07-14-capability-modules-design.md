# CodeReview Agent 能力模块化设计规格

## 1. 目标

将当前固定的 Spring Boot + MySQL + Nacos + Vue 审查能力重构为可注册的能力模块，同时加入保守的通用范型模块。

目标是让后续 Go、C++ 等技术栈以新增模块的形式扩展，而不修改扫描入口、模型调用、结果治理、二次复核或报告流程。

## 2. 模块选择策略

每次审查先由能力注册中心对项目文件运行模块识别：

```text
扫描文件
  -> 能力模块识别、评分与选择
  -> 模块声明覆盖文件
  -> 各模块独立分析并生成审查任务
  -> 统一执行、去重、分级、复核与输出
```

模块必须返回匹配分数和理由。SpringVue 模块通过 Spring Boot 标记、MyBatis、Nacos、Vue、SQL 等事实获得高匹配分数；Generic 模块始终可用，作为兜底模块。

当 SpringVue 模块被选择时，它优先接管 Java、Vue、JavaScript/TypeScript、SQL、MyBatis XML 和 Spring/Nacos YAML 文件。Generic 模块只接收未被专属模块接管的代码文件，避免重复审查、重复 Token 消耗和重复结论。

当没有任何专属模块达到选择阈值，或专属模块分析失败时，Generic 模块独立工作。终端必须说明深度模块没有启用或已降级。

## 3. 统一能力模块接口

每个模块实现以下职责：

```text
detect(files)        # 返回匹配分数与依据
claim_files(files)   # 声明接管的文件
analyze(context)     # 提取事实、关系与静态风险线索
plan_review(context) # 生成带模块边界的审查任务
```

模块分析输出不能直接耦合模型提供方。它只生成统一项目事实、关系、审查任务和覆盖信息；DeepSeek/Ollama 客户端、结果去重、严重问题复核、Token 统计和 Markdown 输出继续由主流程统一处理。

## 4. SpringVue 专属模块

现有 Spring Boot + MySQL + Nacos + Vue 逻辑迁移到 `SpringVueCapability`，保留当前能力：

- Controller、Service、Mapper、SQL、Vue API、Nacos/YAML 的关系提取。
- 接口安全、事务、MyBatis/MySQL、配置、Vue/前后端契约、结构性能任务。
- 风险优先排序、功能切片和专属审查提示词边界。

迁移后，对现有目标项目的检测、任务数量、审查边界和结果分级保持等价，避免模块化重构改变已验证能力。

## 5. Generic 范型模块

Generic 模块用于能力库未覆盖的代码，采用保守方法论：

- 识别常见代码扩展名、构建清单和入口线索，但不声称理解未知框架。
- 分析通用结构、硬编码敏感信息、明显危险字符串拼接、异常处理、资源生命周期、超长函数和重复逻辑线索。
- 将代码按文件职责和大小切片，向模型提供通用证据约束。
- 只报告可从提供上下文证明的问题；语言或框架特定结论必须标记为“需人工确认”。

首版 Generic 模块可识别但不深度理解 C/C++、Go、Python、Java、JavaScript/TypeScript 等常见源文件，为后续专属模块留下扩展入口。

## 6. 注册中心与覆盖信息

注册中心负责：

- 注册内置和未来外部能力模块。
- 收集识别结果，选择专属模块并始终保留必要的 Generic 兜底。
- 按模块声明维护文件所有权，冲突时专属模块优先。
- 汇总模块匹配依据、已覆盖文件和未覆盖文件。

终端摘要增加“已启用能力模块”和“覆盖范围”信息，例如：

```text
已启用能力模块：SpringVue（专属），Generic（补充）
SpringVue 覆盖：105 个文件
Generic 补充覆盖：2 个未归属代码文件
```

## 7. 可靠性与兼容性

- 单一模块分析失败不得阻断其他模块或整个审查流程。
- 无专属模块项目仍可完成 Generic 审查。
- 模块不得直接读取模型 Key 或调用模型服务。
- 模块输出使用现有统一类型，保持结果治理、JSON 复核和报告兼容。
- 文件未被任何模块接管时必须记录为未覆盖范围，不能静默忽略。

## 8. 测试与验收

重构后新增测试：

1. 典型 SpringVue 项目优先选择 SpringVue 模块，并保持现有任务域。
2. 混合项目中专属模块与 Generic 模块的文件覆盖不重叠。
3. 未知技术栈只选择 Generic 模块，并生成保守任务。
4. SpringVue 模块失败时，注册中心降级到 Generic 并记录原因。
5. 结果去重、严重问题二次复核与 Markdown 输出继续通过现有测试。

验收标准：新增能力模块不需要修改主审查工作流；SpringVue 项目保持深度审查；未知项目可以得到明确范围、低误报的基础审查；终端可说明模块选择和覆盖情况。
