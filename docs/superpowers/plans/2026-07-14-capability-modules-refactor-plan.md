# 能力模块化重构计划

1. 新建 `capabilities` 包，定义能力模块接口、识别结果、模块分析结果和注册中心。
2. 将现有 Spring Boot + MySQL + Nacos + Vue 的项目识别、关系提取和任务规划迁移到 SpringVue 模块，保持现有审查域。
3. 新建 Generic 模块：识别未归属代码文件，生成保守的基础安全、可靠性和结构审查任务。
4. 调整主流程：扫描后通过注册中心选择模块、分配文件、汇总模块分析与任务，并显示模块覆盖信息。
5. 保持现有 DeepSeek/Ollama、结果治理、严重问题复核、终端和 Markdown 输出接口不变。
6. 新增模块选择、混合覆盖、未知项目和降级行为测试；运行完整回归测试。
